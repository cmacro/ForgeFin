# ForgeFin Financial Workflows

## Principle

UI must follow accounting workflow.

Never design screens around visual appearance alone.

---

# Voucher Workflow

Draft
→ Review
→ Approve
→ Post
→ Archive

Required actions:

* Save
* Submit
* Approve
* Reject
* Post

---

# Accounts Receivable

Required views:

* Customer Balance
* Aging Analysis
* Outstanding Invoices
* Payment History

---

# Accounts Payable

Required views:

* Vendor Balance
* Aging Analysis
* Unpaid Bills
* Payment Schedule

---

# General Ledger

Required:

* Date Filter
* Account Filter
* Voucher Drilldown
* Export

---

# Financial Reports

Required:

* Balance Sheet
* Income Statement
* Cash Flow
* Trial Balance

---

# Audit Trail

Every financial action should expose:

* Operator
* Time
* Action
* Source

---

# Review Checklist

* Matches accounting workflow?
* Supports audit requirements?
* Supports approval process?
* Provides required financial views?
