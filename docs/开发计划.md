## Phase 0：基础设施（Pre-MVP）

建立支撑所有业务模块的底层能力。

```
优先级: ★★★★★
```

| 任务 | 说明 |
|------|------|
| 0.1 | SQLite 数据层 —— 引入 `rusqlite` + `r2d2` 连接池，建立 `src/db/` 模块 |
| 0.2 | Tauri 命令层 —— 在 `src-tauri/src/` 中建立 commands 模块，Tauri invoke 通信模式 |
| 0.3 | 数据库 Schema —— 会计科目表、凭证表、凭证分录表、客户/供应商表、公司表 |
| 0.4 | 认证系统 —— 登录页 + 用户管理 |
| 0.5 | 公司/账套管理 —— 多组织支持 |

## 迭代节奏

每 1-2 周完成 1 个小功能 → 立即在真实业务中使用 → 收集反馈 → 优化。

**最优先启动**：Phase 0 (数据层 + 认证) → Phase 1 (科目 + 凭证录入 + 凭证查询)。
