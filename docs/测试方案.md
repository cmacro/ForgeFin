# ForgeFin 测试方案

## 现状

项目目前无任何测试基础设施：
- 无 `#[test]` / `#[cfg(test)]` 代码块
- 无 `tests/` 目录
- 无 `[dev-dependencies]`
- 无 CI 配置
- 无 Makefile test 目标

---

## 一、后端测试（`src-tauri/` crate）

### 核心策略

利用 `rusqlite` 的 `:memory:` 模式，为每个测试创建独立内存数据库，测试间完全隔离。

### 依赖（`src-tauri/Cargo.toml` 的 `[dev-dependencies]`）

| Crate | 用途 |
|-------|------|
| `rstest` | 参数化测试、fixture 管理 |
| `tempfile` | 临时文件/目录（备份测试需要） |
| `mockall` | mock Tauri 的 `AppHandle` / `State` |

### 测试结构

```
src-tauri/
  src/
    db/
      schema.rs          -- 已有 schema 初始化
      mod.rs             -- 加 #[cfg(test)] 模块，提供 test_db() 工厂函数
    commands/
      account.rs         -- 每个命令文件底部加 #[cfg(test)] mod tests
      voucher.rs
      ...
  tests/                 -- 集成测试目录
    common/
      mod.rs             -- 共享的测试辅助函数
    accounts_test.rs
    vouchers_test.rs
```

### 关键模式

```rust
// db/mod.rs — 测试用工厂函数
#[cfg(test)]
pub fn test_db() -> rusqlite::Result<Connection> {
    let conn = Connection::open_in_memory()?;
    schema::initialize_system_db(&conn)?;
    Ok(conn)
}

// commands/account.rs — 单元测试
#[cfg(test)]
mod tests {
    use super::*;
    use crate::db::test_db;
    use rstest::rstest;

    #[rstest]
    #[case("valid_name", true)]
    #[case("", false)]
    fn test_validate_account_name(#[case] name: &str, #[case] expected: bool) {
        assert_eq!(validate_account_name(name), expected);
    }

    #[test]
    fn test_create_account() {
        let conn = test_db().unwrap();
        let result = create_account_internal(&conn, "Cash", "1001", "asset");
        assert!(result.is_ok());
    }
}
```

### 测试优先级

| 层级 | 优先级 | 说明 |
|------|--------|------|
| 数据库层（`db/`） | **最高** | schema 初始化、CRUD、事务回滚、并发 |
| 命令层（`commands/`） | **高** | 输入验证、业务逻辑（过账、审计）、错误处理 |
| 工具函数 | **中** | 金额格式化、日期处理、权限校验 |

## 二、前端测试（`forgefin-ui` crate）

### 分层策略

| 层级 | 工具 | 测试内容 |
|------|------|---------|
| 纯逻辑 | 标准 `#[test]` | 金额格式化、日期处理、状态机逻辑 |
| 组件渲染 | `wasm-bindgen-test` | 组件挂载、信号响应、DOM 断言 |
| E2E | `tauri-driver` / WebDriver | 完整用户流程 |

### 依赖（`Cargo.toml` 的 `[dev-dependencies]`）

| Crate | 用途 |
|-------|------|
| `wasm-bindgen-test` | WASM 环境下的测试运行器 |
| `rstest` | 参数化测试 |
| `gloo` | DOM 操作/事件模拟 |

### 测试结构

```
src/
  lib.rs                -- 加 #[cfg(test)] 模块
  utils.rs              -- 纯逻辑函数，底部加测试模块
  components/
    form/
      search_form.rs    -- 组件测试
    table/
      pagination.rs    -- 组件测试
```

### 关键模式

```rust
// 纯逻辑测试（无需 WASM 环境）
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_format_currency() {
        assert_eq!(format_currency(1234.50), "1,234.50");
    }
}

// 组件测试（需要 wasm-bindgen-test）
#[cfg(test)]
#[wasm_bindgen_test::wasm_bindgen_test]
fn test_pagination_renders() {
    // 挂载组件，断言 DOM 结构
}
```

## 三、基础设施

### Makefile 添加

```makefile
test:
	cargo test --workspace

test-backend:
	cargo test -p forgefin

test-frontend:
	cargo test -p forgefin-ui
```

### CI（GitHub Actions）

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test --workspace
      - run: cargo clippy -- -D warnings
      - run: cargo fmt --check
```

## 四、实施步骤

1. 添加 `[dev-dependencies]` 到两个 `Cargo.toml`
2. 在 `db/mod.rs` 中实现 `test_db()` 工厂函数
3. 为数据库层（schema、CRUD）编写单元测试
4. 为命令层（account、voucher）编写单元测试
5. 为前端纯逻辑函数编写测试
6. 添加 Makefile test 目标
7. 配置 GitHub Actions CI
8. 逐步补充集成测试和组件测试
