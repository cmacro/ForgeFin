## 三、关键设计决策

### 1. 数据库架构
- **系统库** `forgefin_system.db`：users / companies / user_company_permissions（跨公司共享）
- **公司库** `forgefin_company_{uuid}.db`：accounts / contacts / vouchers / voucher_entries / voucher_audit_logs
- 数据目录：`dirs::data_dir()/ForgeFin/`（companies/、backups/ 子目录）
- 连接：`Mutex<Option<Connection>>`（系统）+ `Mutex<HashMap<String, Connection>>`（公司）
- PRAGMA：`foreign_keys=ON`、`journal_mode=WAL`、`synchronous=NORMAL`

### 2. 金额存储
- 用 **分（i64）的字符串** 存储金额（如 "12500" 表示 125.00 元）
- 前端显示需除以 100 格式化（当前前端直接显示分值，**待优化**）
- 凭证保存前 `sum_entries` 校验借贷平衡

### 3. 会话模型
- 桌面应用进程内单用户会话（`SessionState` Mutex）
- 前端 `auth.rs` 用 `RwSignal` 全局信号镜像后端会话
- 启动时 `Session::init()` 调 `current_user_cmd` 恢复会话

### 4. IPC 通信
- 不引入 `tauri` Rust crate 到前端（避免重依赖）
- 通过 `window.__TAURI__.core.invoke`（tauri.conf.json `withGlobalTauri:true`）
- 用 `js_sys::Reflect::get` 动态查找 invoke 函数

## 四、遇到的问题与修复

### 问题 1：`OptionalExtension` 缺失
**现象**：`stmt.query_row(...).optional()` 报错 "no method named optional"
**原因**：未 import `rusqlite::OptionalExtension`
**修复**：`use rusqlite::{params, Connection, OptionalExtension};`

### 问题 2：`Params` trait 未实现
**现象**：`&mut [&dyn ToSql]: Params` not satisfied（list_vouchers 动态参数）
**原因**：rusqlite 0.40 的 `Params` 只对 `&[&dyn ToSql]` 实现，不对 `&mut [&dyn ToSql]`
**修复**：改用 `rusqlite::params_from_iter`：
```rust
let refs: Vec<&dyn rusqlite::ToSql> = p.iter().map(|b| b.as_ref()).collect();
conn.query_row(&sql, params_from_iter(refs.iter()), |row| row.get(0))
```

### 问题 3：`?` 无法转换 `rusqlite::Error` 为 `String`
**现象**：`stmt.query_map(...)? ` 报错 `From<rusqlite::Error>` 未实现 for String
**原因**：孤儿规则下无法为 `String` impl `From<rusqlite::Error>`
**修复**：所有 `query_map(...)? ` 改为 `.map_err(|e| format!("...失败: {e}"))?`

### 问题 4：`let _ = guard.company(...)?` 立即释放锁
**现象**：`non-binding let on a synchronization lock`（deny 级别）
**原因**：`let _ = ` 会立即 drop MutexGuard，惰性初始化虽完成但锁立即释放（语义上没错，但 lint 拒绝）
**修复**：改为 `let _guard = guard.company(&company_id)?;`（保活到作用域结束）

### 问题 5：命令注册位置错误
**现象**：`__tauri_command_name_user_companies_cmd not found in company`
**原因**：lib.rs 把 `user_companies_cmd` 注册在 `company::` 下，实际在 `user::` 模块
**修复**：改 `company::user_companies_cmd` → `user::user_companies_cmd`

### 问题 6：未使用的 mut 警告
**现象**：`unused_mut: mut companies_guard`（list 操作不需可变引用）
**修复**：只读命令（list_accounts/list_contacts/list_vouchers）去掉 `mut`；写命令（create/update/delete）保留 `mut`（需 `get_mut`）

### 问题 7：lifetime 警告
**现象**：`hiding a lifetime that is elided elsewhere is confusing`
**修复**：`MutexGuard<Option<Connection>>` → `MutexGuard<'_, Option<Connection>>`

### 问题 8：dead_code 警告
**修复**：
- `db_state` 辅助函数加 `#[allow(dead_code)]`
- `find_user_by_id` 加 `#[allow(dead_code)]`
- 移除 `Session::current_company` 方法
- 移除未使用的 `SystemTime` import、`HashMap` import

### 优先级 2：金额显示格式化
当前前端直接显示分值（如 "12500"），应改为元（"125.00"）。建议在 `ipc.rs` 或新建 `format.rs`：
```rust
pub fn fmt_money(cents: &str) -> String {
    cents.parse::<i64>().ok()
        .map(|c| format!("{:>2}.{:02}", c/100, c.abs()%100))
        .unwrap_or_else(|| cents.to_string())
}
```

### 优先级 3：首次登录引导
后端无内置 admin 用户。需在 `lib.rs` 启动时检查 users 表为空则创建默认 admin：
```rust
// run() 中 DbState 初始化后
let guard = db_state.lock()?;
let sys = guard.system()?;
let conn = sys.as_ref().unwrap();
if system::list_users(conn)?.is_empty() {
    system::create_user(conn, "admin", "管理员", "admin123", None, true)?;
}
```

### 优先级 4：用户管理 UI 完善
`settings.rs` 的用户管理区当前是占位文字。需接入 `list_users_cmd`/`create_user_cmd`/`grant_permission_cmd`。

### 优先级 5：导航激活逻辑
`MainShell` 中 `Session::has_company()` 在 `children` 闭包外读取，可能不响应式。需用 `move ||` 闭包包裹：
```rust
let children = move || {
    if !has_company.get() { return ...; }
    ...
};
```
当前写法 `has_company.get()` 在闭包外求值一次，**这是 bug**，应移入 `move ||` 内。

### 优先级 6：凭证录入页的账号选择
`voucher_entry.rs` 中科目 `<select>` 用 `e.account_id == id` 判断 selected，但 `e` 是闭包外捕获的值，可能不更新。建议用 `prop:value` 或重渲染。
