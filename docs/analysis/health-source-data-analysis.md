# 健康管理公司A数据来源分析

## 1. 数据源概述

| 数据源 | 文件名 | 来源 | 更新频率 | 用途 |
|--------|--------|------|----------|------|
| 银行流水 | `银行流水.tsv` | 银行对公账户导出 | 每日/次日 | 现金日记账、银行对账 |
| 订单流水 | `订单流水.tsv` | 银行工商商户网站导出 | 按需 | 手续费拆分、收入确认 |
| 数据汇总 | `数据汇总.tsv` | 人工根据上述数据整理 | 每日/按需 | 凭证录入直接依据 |

## 2. 字段可用性分析

## 3. 数据特征

### 3.1 业务类型

健康管理公司主营业务：

| 业务类型 | 示例 | 说明 |
|----------|------|------|
| 产康服务 | 客户A产康充值 | 产后康复服务 |
| 美容服务 | 客户B牛奶光、客户C美肤炫 | 皮肤护理/美容 |
| 身体护理 | 客户D服务 | 进口美容仪器服务 |
| 产品销售 | 货款（供应商A） | 美容产品 |
| 场地成本 | 水电费/房费 | 房租、水电 |

### 3.2 交易流水特征

- **POS 收入为主**：客户通过银行卡、微信、支付宝支付
- **T+1 清算**：银行次日清算，订单流水与银行流水日期相差1天
- **手续费实时扣除**：银行直接扣除，订单流水中记录实收金额
- **单日多笔合并**：银行流水单笔清算金额为多笔订单合计

## 6. 数据质量问题

1. **银行流水 POS 清算合并**：单日多笔 POS 交易合并为一笔银行转入
2. **2026-07-01 清算差异**：订单合计 16,340.00 与银行流水 16,299.15 存在 0.45 元差异
3. **微信备注依赖**：事由信息需从微信群获取，非自动导出
4. **借记卡/信用卡列无值**：实际交易通过微信/支付宝渠道完成

## 9. 结论

健康管理公司（样本企业A）的数据特征：

- **交易频繁、单笔金额小**：POS 收入为主
- **手续费透明**：订单流水中完整记录费率
- **T+1 清算**：需次日核对银行流水
- **微信备注重要**：客户来源、事由依赖人工整理

系统应优先实现订单流水导入、银行流水自动勾稽、差异提示、和基于核对结果的凭证生成功能。
