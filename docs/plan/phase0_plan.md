# Phase 0: 基础设施（Pre‑MVP）

## 目标
建立支撑所有业务模块的底层能力。

## 任务列表

- **0.1 SQLite 数据层**
  - 在 `src-tauri/src/db/` 建立数据库模块。
  - 使用 `rusqlite` 的 `Connection`，每公司一个数据库实例，用 `Mutex` 管理线程安全。
  - 不引入连接池，保持精简。

- **0.2 Tauri 命令层**
  - 在 `src-tauri/src/commands/` 中建立命令模块，采用 Tauri `invoke` 通信模式。
  - 所有命令需携带当前公司 ID，动态切换数据库实例。

- **0.3 数据库 Schema**
  - **系统库**：用户、公司/账套注册信息。
  - **业务库（每公司）**：会计科目表、凭证表、凭证分录表、客户/供应商表等。

- **0.4 认证系统**
  - 登录页 + 用户管理。
  - 用户可跨公司/账套，权限按公司独立控制。

- **0.5 公司/账套管理**
  - 多组织支持，每个公司独立数据库文件，业务数据完全隔离。
  - 支持新建、切换、编辑公司信息。

- **0.6 备份与恢复**
  - 支持手动/自动备份公司数据库文件，支持从备份文件恢复。
  - 备份粒度：单公司 / 全系统。
