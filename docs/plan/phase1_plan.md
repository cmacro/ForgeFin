# Phase 1: 凭证核心（MVP 核心）

## 目标
让真实业务能跑起来——从手工录入到打印。

## 任务列表

- **1.1 会计科目管理**
  - 科目 CRUD + 树形列表（替换 Placeholder）。
  - 科目按公司隔离，每个公司独立一套科目表。

- **1.2 客户/供应商管理**
  - 基础信息管理（增删改查）。
  - 客户/供应商按公司隔离，互不共享。

- **1.3 凭证录入页**
  - 新增凭证表单（日期、摘要、科目选择、借贷金额、自动平衡）。
  - 凭证归属当前公司数据库。

- **1.4 凭证保存**
  - 通过 Tauri 命令写入当前公司的 SQLite 数据库文件。

- **1.5 凭证查询**
  - 按期间/字号/类型/状态过滤 + 分页查询。
  - 查询范围限定在当前公司。

- **1.6 凭证审核/反审核**
  - 状态流转 + 审核日志记录。

- **1.7 凭证打印**
  - 打印模板（PDF/打印机）。
