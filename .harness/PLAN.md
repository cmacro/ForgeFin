# ForgeFin 开发计划

本计划将系统开发分为六个阶段。每个阶段完成后必须经过人工验收方可进入下一阶段。

## 阶段 1：基础设施与基础模型
- [ ] 搭建 Tauri 2.x + Leptos 前端环境
- [ ] 设计并实现 SQLite 基础数据库 Schema（项目表、基础配置表）
- [ ] 实现基础的项目导航与概览界面
- **验收点**：能够启动空项目，在界面上看到项目列表且数据库连接正常。

---

## 阶段 2：核心项目收支核算与汇总报表 (模块 1, 10)
- [ ] 实现项目级收支明细记录（凭证号、摘要、收入/支出各项）
- [ ] 实现项目利润实时计算逻辑 (`收入合计 - 支出合计 = 项目利润`)
- [ ] 实现年度汇总报表，支持跨项目数据统计
- **验收点**：能够为特定项目录入收支凭证，且汇总表能正确计算项目利润。

---

## 阶段 3：资金流转与发票管理 (模块 7, 8)
- [ ] 实现付款/收款账户管理（银行账户、交易单位）
- [ ] 实现资金流水记录与批量继承逻辑
- [ ] 实现发票状态跟踪（专票/普票、已收/后补）
- **验收点**：资金流水能与项目收支记录关联，发票状态可正确标记。

---

## 阶段 4：资源与合同管理 (模块 3, 4, 5)
- [ ] 实现材料采购跟踪（供应商、金额、余款）
- [ ] 实现劳务分包管理（班组结算、已付/未付跟踪）
- [ ] 实现保证金与押金全生命周期跟踪（投标/履约/退还）
- **验收点**：能够独立管理材料与劳务合同，且保证金状态可追踪。

---

## 阶段 5：税务、报销与废旧物资管理 (模块 2, 6, 9)
- [ ] 实现项目预缴税费管理（增值税、城建税、教育费等）
- [ ] 实现费用报销系统（差旅、办公、房租水电等）
- [ ] 实现废旧物资回购记录
- **验收点**：税务预缴能正确归集到项目成本，报销流程闭环。

---

## 阶段 6：系统集成与最终优化
- [ ] 全流程数据一致性校验
- [ ] SQLite 索引优化与性能调优
- [ ] 整体 UI/UX 润色与最终 UAT 测试
- **验收点**：通过全场景业务走通测试，系统运行流畅。
