## Phase 0：基础设施 (Pre-MVP) ★★★★★

> 目标：建立支撑所有业务模块的底层能力。优先启动。

- [ ] **0.1** SQLite 数据层 — `src-tauri/src/db/`，rusqlite `Connection` + `Mutex` 线程安全，每公司一实例，不引入连接池
- [ ] **0.2** Tauri 命令层 — `src-tauri/src/commands/`，invoke 通信，所有命令携带 company_id 动态切换数据库实例
- [ ] **0.3** 数据库 Schema — 系统库 `forgefin_system.db`（用户、公司/账套注册）+ 公司库 `forgefin_company_{id}.db`（科目表、凭证表、凭证分录表、客户/供应商表等）
- [ ] **0.4** 认证系统 — 登录页 + 用户管理（用户跨公司，权限按公司独立控制）
- [ ] **0.5** 公司/账套管理 — 多组织支持，新建/切换/编辑公司，业务数据完全隔离
- [ ] **0.6** 备份与恢复 — 手动/自动备份公司数据库文件，从备份恢复，粒度：单公司/全系统，恢复需二次确认

## 目录约定速查

| 类型 | 路径 |
|------|------|
| 页面 | `src/pages/*.rs` |
| 组件 | `src/components/{layout,table,form,charts}/` |
| 数据层 | `src-tauri/src/db/` |
| Tauri 命令 | `src-tauri/src/commands/` |

## 完成校验

每个任务完成前必须通过：

```sh
cargo fmt
cargo check
```

两者均成功方可标记为 `[x]`。

---

## 迭代节奏

每 1–2 周完成 1 个小功能 → 真实业务使用 → 收集反馈 → 优化。
最优先启动：**Phase 0 (数据层 + 认证) → Phase 1 (科目 + 凭证录入 + 凭证查询)**。
