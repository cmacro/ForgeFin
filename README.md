# ForgeFin

财务工作助手（Financial Workspace）

## 产品目标

让每个财务人员每天节省 1～3 小时。

## 技术栈

| 层 | 技术 |
| --- | --- |
| 前端框架 | Leptos 0.8 |
| 桌面容器 | Tauri 2 |
| 样式 | Tailwind CSS v4 |
| 数据库 | SQLite（每公司独立 `.db` 文件，多账套隔离） |
| 语言 | Rust Stable |

## 产品路线图

### 第一阶段 · 效率工具

从一个人每天的重复工作中发现痛点，用软件替代手工操作。

一个典型的财务工作日流程：

```
打开微信 → 下载附件 → 下载银行流水 → 打开Excel → 录入凭证 → 复制摘要 →
填写金额 → 录入科目 → 保存 → 生成凭证 → 打印 → 对账 → 生成报表
```

真正耗时间的不是会计知识，而是：

- 不停复制粘贴
- 导入导出数据
- 找数据、查余额
- 查科目、查客户、查供应商
- 查发票、下载附件
- 手动 OCR 录入

### 第二阶段 · 自动化

| 能力 | 说明 |
| ---- | ---- |
| OCR 发票识别 | 扫描发票 → 自动提取日期、金额、税率、供应商、发票号 |
| 银行流水导入 | CSV/Excel 直接导入，自动生成收付款记录 |
| 自动匹配 | 银行流水自动匹配销售订单/应收应付 |
| 自动生成凭证 | 根据业务模板（如采购入库）自动产生借贷分录 |
| AI 辅助 | 批量上传发票 → AI 识别 → 自动生成凭证 → 人工审核 |

### 第三阶段 · 完整财务系统

```
基础资料 → 客户 / 供应商 / 员工 / 科目 / 项目 / 部门
     ↓
凭证 → 总账 → 明细账 → 资产负债表 / 利润表 / 现金流
```

### 第四阶段 · 产品化

面向中小企业和代账人员的智能财务软件。

**目标用户**：个人会计 / 小公司 / 财务代理公司 / 工作室 / 电商卖家 / 自由职业财务人员

## 软件架构

### 多公司/多账套架构

每个公司/账套拥有独立的 SQLite 数据库文件，业务数据完全隔离互不干扰：

```
┌─────────────────────────────────────┐
│         forgefin_system.db          │  ← 系统库：用户、公司注册信息
│  (用户表、公司表、用户-公司关联)       │
└─────────────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
┌──────────┐  ┌──────────┐  ┌──────────┐
│ company_1 │  │ company_2 │  │ company_3 │  ...  ← 各公司独立业务库
│   .db     │  │   .db     │  │   .db     │
│ 科目/凭证  │  │ 科目/凭证  │  │ 科目/凭证  │
│ 客户/供应商 │  │ 客户/供应商 │  │ 客户/供应商 │
│  报表数据  │  │  报表数据  │  │  报表数据  │
└──────────┘  └──────────┘  └──────────┘
```

- 用户登录后选择公司，所有业务操作自动路由到对应数据库
- 不引入 ORM，使用 `rusqlite` 直接操作，保持精简
- 连接池 (`r2d2`) 按需管理每个数据库文件的连接

### 工作流

按工作流组织：

```
收集资料 → 识别 → 分类 → 生成凭证 → 审核 → 记账 → 报表 → 报税
```

每一步均可加入 AI 辅助。

## 开发理念

1. **先记录，后编码** — 花时间记录每天的工作流程和真实痛点
2. **小步快跑** — 每完成一个小功能立即在真实工作中使用，不等到系统开发完成
3. **持续度量** — 记录每一次操作的点击次数、耗时和易错点，持续优化
4. **从个人到通用** — 功能成熟后再抽象为通用能力，逐步服务更多财务人员

> 不是功能比金蝶更多，而是让一个财务人员每天节省 1～3 小时。
>
> 如果真正做到这一点，即使第一版只有金蝶 10% 的功能，它对目标用户的价值也可能远远超过一个功能齐全但操作复杂的系统。
